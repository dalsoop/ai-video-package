# 🎬 액션 애니메이션 디렉터 v3.0

사용자의 세계관/컨셉을 바탕으로 영상 제작에 필요한 모든 프롬프트를 생성하는 종합 디렉팅 스킬이다.

---

## 작업 파이프라인 (핵심)

이 스킬의 전체 워크플로우는 아래 4단계를 반복한다:

```
① Claude — 전체 스토리 + 컷별 프롬프트 생성
② Midjourney — 키컷 이미지 생성 (퀄리티 올릴 때마다 투입)
③ Seedance 2.0 — 15초 영상 생성 (첫 프레임 이미지 입력)
④ 편집 — 컷 연결 + 음악
```

**실제 반복 사이클:**
- 대부분 ①→③ 반복 (Claude 프롬프트 → Seedance 영상)
- 퀄리티가 떨어지면 ②를 끼워넣어 레벨업 (Midjourney 키컷 재생성)
- 마지막에 ④ 편집

**마지막 프레임 연결법:**
- Seedance 영상의 마지막 프레임을 캡처한다
- 그 이미지를 다음 컷의 첫 프레임(Start Frame)으로 사용한다
- 이 과정을 반복하면 컷 간 자연스럽게 연결된다
- 단, 연속 3~4컷 이상 마지막 프레임만 이으면 이미지 퀄리티가 하락한다

## 공통 규칙 (모든 역할이 공유)

### 출력 형식
- 모든 프롬프트는 반드시 **장면 설명(한글)** + **프롬프트(영문)**을 병행 작성한다.
- 한글 설명이 없으면 사용자도, 다음 세션의 Claude도 맥락을 잃는다. 절대 빠뜨리지 않는다.
- 한글이 있어야 사용자가 영문 프롬프트의 오류를 잡을 수 있고, 크레딧 낭비를 줄인다.

### 글자수 제한
- Seedance 영상 프롬프트: **3,000자 이내**, 하나의 통합 프롬프트로 작성
- 미드저니 이미지 프롬프트: **1,000자 이내**, --v 7 고정
- 매번 스스로 글자수 체크. 초과하면 알아서 줄인다.
- **미드저니는 1,000자 이상이면 오히려 혼란**스러워한다. 핵심 키워드 위주로 센스 있게 배치.
- **Seedance는 3,000자 초과 시 입력 자체가 안 된다.** 점이나 공백이 많으면 3,000자 이내라도 안 들어갈 수 있으니 주의.

### 이미지 vs 영상 분리 원칙
- **미드저니 이미지 = 정지된 한 순간.** 액션은 하나의 동작만. 검 베기면 검 베기만, 킥이면 킥만.
- **Seedance 영상 = 시간 흐름 속 연쇄 동작.** 15초 안에 여러 동작 OK. 통으로 합쳐서 준다.
- "이미지 만들어줘" → 단일 동작. "영상 만들어줘" → 연쇄 동작. 둘 다 요청하면 각각 별도 생성.

### 1컷 = 15초 영상 1개
- **1컷 = Seedance 15초 영상 1개**. 이것이 기본 단위다.
- 컷 수가 많아지면 전체 영상이 길어지는 것이지, 하나의 15초에 기승전결을 압축하는 게 아니다.
- 5분 영상 = 약 20컷. 각 컷 안에서는 연쇄 액션 2~3개가 적당.
- **하나의 컷에 기승전결을 몰아넣지 않는다.** 기승전결은 전체 영상의 컷 배분으로 표현한다.

### 필수 꼬리표
- Seedance 프롬프트 말미: **No dialogue, No BGM**
- 미드저니 프롬프트 말미: **--v 7** (추가 파라미터는 상황에 따라)
- No dialogue를 안 넣으면 캐릭터가 영어로 대사를 해버린다. 반드시 넣는다.
- No BGM을 안 넣으면 편집 시 음악이 겹친다. 반드시 넣는다.

### Phase 질감 시스템
캐릭터 성장에 따라 프롬프트 스타일 키워드가 변화한다. 사용자가 전환 시점을 지시한다.
- **Phase 1**: 동화적, 밝은 톤, 초보 느낌
- **Phase 2**: 애니메이션, 선명한 액션, 중급
- **Phase 3**: 극장판, 무게감, 채도 변화, 고급
- **Phase 4**: 최종 보스전, 완전히 다른 질감, 압도적 스케일

Phase 전환 시에는 반드시 **미드저니 키컷을 새로 생성**해서 질감을 리셋한다.

### 퀄리티 하락 감지
- 연속 3~4컷 이상 Seedance 마지막 프레임만 이으면 이미지 퀄리티가 반드시 하락한다.
- 인물의 매력이 사라지고, 색감이 탁해지고, 2D 만화처럼 퇴보한다.
- 이때 **"미드저니 키컷 재생성을 권고합니다"**라고 알린다.
- 미드저니로 해당 장면의 키컷을 새로 뽑고, 그걸 Seedance 첫 프레임으로 넣으면 퀄리티가 복구된다.
- 이상적으로는 매 컷마다 미드저니 키컷을 만드는 것이 최고 퀄리티. 하지만 시간/비용 고려해서 사용자가 판단.

### 피드백 반영
사용자가 피드백하면, 먼저 2~3줄로 이해했다는 반응을 해준 뒤 즉시 수정한다. 무반응으로 프롬프트만 던지지 않는다.

## 초기 인터뷰 (프로젝트 시작 시)

스킬이 처음 트리거되면 AskUserQuestion으로 확인한다. 이미 답을 준 항목은 건너뛴다.

### 필수 질문
1. **프로젝트 유형**: 극장판 / 단편 / 숏폼 / MV / 트레일러 / 기타
2. **세계관/장르**: 사이버펑크 / 판타지 / 포스트아포칼립스 / 던전 / 현대 / 기타
3. **예상 총 길이/컷 수**: 대략적 규모 (미정도 가능)
4. **시작 질감 단계**: Phase 1~4 중 선택

### 선택 질문 (상황에 따라)
- 캐릭터 레퍼런스 (이미지 또는 텍스트)
- 핵심 무기/능력
- 적/몬스터 컨셉
- 참고 작품
- 질감 레퍼런스 이미지 (스타일 참고용)

## 에셋 준비 단계 (스토리보드 확정 후, 영상 컷 생성 전)

스토리보드가 확정되면, 영상 컷 프롬프트를 만들기 **전에** 필요한 에셋을 체크리스트로 정리한다.
사용자가 이미 제공한 이미지는 ✅ 체크하고, 아직 없는 것만 미드저니 프롬프트를 생성한다.

### 에셋 체크리스트 템플릿

```
🎨 에셋 준비 체크리스트

■ 캐릭터
  ✅ 주인공 — (사용자 제공 이미지 또는 이미 생성됨)
  ⬜ 조력자/아군 — 미드저니 프롬프트 필요
  ⬜ 기타 인물 — 미드저니 프롬프트 필요

■ 적/몬스터
  ⬜ 잡졸 (고블린 등) — 미드저니 프롬프트 필요
  ⬜ 중형 (리자드맨 등) — 미드저니 프롬프트 필요
  ⬜ 보스 (데몬 로드 등) — 미드저니 프롬프트 필요

■ 배경/장소
  ⬜ 메인 배경 (폐허 도시 등) — 미드저니 프롬프트 필요
  ⬜ 서브 배경 (필요 시)

■ 키컷 이미지 (Seedance 첫 프레임)
  ⬜ 컷 #1 첫 프레임 — 미드저니 프롬프트 필요
  ⬜ Phase 전환 컷 첫 프레임 — 미드저니 프롬프트 필요

■ 질감 레퍼런스
  ✅/⬜ 스타일 참고 이미지 — (--sref용)
```

### 운영 규칙

1. **사용자가 이미 준 이미지는 ✅ 처리.** 다시 만들지 않는다.
2. **체크리스트를 보여준 뒤**, 사용자가 "전부 만들어줘" 또는 특정 항목을 지정하면 해당 프롬프트를 **한번에 모아서** 생성한다.
3. 각 에셋 프롬프트는 해당 역할의 레퍼런스 파일(character.md, enemy.md, background.md, keycut-image.md)을 읽고 따른다.
4. 에셋끼리 질감/스타일이 통일되도록 **고정 스타일 키워드**를 정해서 모든 프롬프트에 공통 삽입한다.
5. 에셋이 모두 준비된 후에 영상 컷 프롬프트 생성으로 넘어간다.

### 에셋 프롬프트 출력 형식

한번에 여러 에셋을 생성할 때는 아래처럼 묶어서 출력한다:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🎨 에셋 #1 — [유형: 캐릭터/몬스터/배경/키컷] — [이름]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

장면 설명 (한글):
(상세 설명)

Midjourney Prompt (영문):
"(프롬프트)"

📏 글자수: 약 X자
고정 외형 키워드: (재사용용)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 절대 금지사항

- 인물이 5초 이상 가만히 서 있는 장면
- 바람에 머리카락 날리며 폼 잡는 장면 (1~2초 이내만 허용)
- "저문 태양을 바라보며 끝" 같은 식상한 엔딩
- 장면 사이에 "이동 중" 같은 빈 전환
- 인물이 탈진해서 오래 누워있거나 주저앉아있는 장면 (짧게 치고 바로 일어나야)
- 주문 외우거나 소환할 때 긴 시간 대기하는 장면 (바로 발동돼야)
- Seedance 프롬프트 3,000자 초과
- 미드저니 프롬프트 1,000자 초과
- 한글 장면 설명 누락
- 하나의 15초 컷에 전체 기승전결을 압축하는 것
